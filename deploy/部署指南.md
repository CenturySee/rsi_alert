# RSI 预警 — 部署指南（Ubuntu/Debian + Cloudflare Tunnel）

把 `rsi-alert` 部署到「有公网 IP 但**无固定 IP**、且运营商**封了 80/443**」的小主机上，
DNS 托管在 **Cloudflare**。

## 为什么用 Cloudflare Tunnel 而不是 DDNS

传统「DDNS + 端口转发 + 反向代理」在你的场景走不通：80/443 被封，外部进不来。
**Cloudflare Tunnel（cloudflared）** 由主机**主动向外**建立加密连接，因此：

- ✅ **不需要任何入站端口** —— 80/443 被封无所谓
- ✅ **不需要 DDNS** —— 公网 IP 怎么变都行，隧道自己保持连接
- ✅ **免费自动 HTTPS** —— Cloudflare 边缘签发证书
- ✅ **隐藏真实公网 IP**，更安全；NAT 后面也能用

```
公网用户 → Cloudflare 边缘(HTTPS) → 加密隧道(出站) → 小主机 cloudflared → 127.0.0.1:8000 (uvicorn)
```

---

## 项目相关的两个点

1. `data/` 缓存目录是相对路径（`__file__/data`），**可移植，无需改**。
2. `opentdx` 是公开 PyPI 包（`requires-python >=3.12`，与本项目一致），**部署时无需拷贝**。
   本地 `pyproject.toml` 为方便开发用 editable 本地源覆盖了它；`deploy.sh` 会在服务器副本上
   去掉这个覆盖、删掉锁文件，让 uv 直接从 PyPI 安装。

## 第 1 步：把代码传到服务器

服务器上（用户名假设 `ubuntu`，部署到 `/opt/quant`）：

```bash
sudo mkdir -p /opt/quant && sudo chown $USER /opt/quant
```

在 **Windows 本机**（git bash），把项目目录传过去（替换 `SERVER_IP`）：

```bash
scp -r /c/xzq/projects/quant/rsi-alert   ubuntu@SERVER_IP:/opt/quant/
```

> `opentdx` 走 PyPI，不用拷。

## 第 2 步：在服务器上跑一键脚本

```bash
cd /opt/quant/rsi-alert/deploy
chmod +x deploy.sh
./deploy.sh
```

脚本会：清理 venv/缓存 → 移除本地 opentdx 源覆盖（改从 PyPI 装）→ 装 uv → `uv sync` →
跑测试 → 安装并启动 `rsi-alert` systemd 服务（绑 `127.0.0.1:8000`）。

完成后验证本地：

```bash
curl -s 127.0.0.1:8000 | head    # 应返回 HTML
```

> opentdx 要连通达信行情服务器（国内）、akshare 取交易日历，确认小主机出站网络通畅。

## 第 3 步：装 cloudflared 并建隧道

> 如果之前在这台小主机上跑过 cloudflared，先按 [`检查已有隧道.md`](检查已有隧道.md)
> 判断能否直接复用，避免重复创建。

```bash
# 安装（amd64；ARM 小主机把 amd64 换成 arm64）
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 \
  -o /tmp/cloudflared
sudo install /tmp/cloudflared /usr/local/bin/cloudflared

# 登录（打印一个 URL，浏览器打开授权你的域名）
cloudflared tunnel login

# 创建隧道，并把子域名解析到隧道（替换成你的域名）
cloudflared tunnel create rsi-alert
cloudflared tunnel route dns rsi-alert rsi.example.com
```

把 `cloudflared-config.example.yml` 拷成 `~/.cloudflared/config.yml`，
填入 `create` 输出的 `<TUNNEL_ID>` 和你的域名，然后：

```bash
sudo cloudflared --config /home/ubuntu/.cloudflared/config.yml service install
sudo systemctl enable --now cloudflared
sudo systemctl status cloudflared
```

## 第 4 步：验证

浏览器打开 **https://rsi.example.com** —— 直接出页面，HTTPS 由 Cloudflare 自动提供。

## 运维备忘

| 操作 | 命令 |
|------|------|
| 看 app 日志 | `journalctl -u rsi-alert -f` |
| 看隧道日志 | `journalctl -u cloudflared -f` |
| 重启 app | `sudo systemctl restart rsi-alert` |
| 更新代码 | 服务器上 `cd /opt/quant/rsi-alert && git pull` → `sudo systemctl restart rsi-alert` |

> **更新代码说明（git clone 部署）**
>
> 服务器是 `git clone` 检出的，常规更新就两步：
> ```bash
> cd /opt/quant/rsi-alert
> git pull
> sudo systemctl restart rsi-alert      # 仅改 .py / 前端时足够
> curl -s 127.0.0.1:8000 | head         # 自检，返回 HTML 即 OK
> ```
> - **只改了 `.py` / `frontend/` 时无需 `uv sync`**；改了 `pyproject.toml`（增删依赖）后才需要先 `uv sync` 再 restart。
> - `deploy.sh` 首次部署时改过本机 `pyproject.toml`（删本地 opentdx 源）并删了 `uv.lock`，这属于工作区本地改动。只要后续提交**不碰这两个文件**，`git pull` 会干净快进、不冲突；若某次提交确实改了它们而 `git pull` 报冲突，用 `git checkout -- pyproject.toml uv.lock` 丢弃服务器本地改动后重新 pull 即可。
> - 前端 `index.html` 更新后，浏览器访问记得 **Ctrl+F5** 强刷清缓存。

## 安全建议（强烈推荐）

隧道默认全公网可访问。这是个人盯盘工具，建议在 **Cloudflare Zero Trust → Access**
给 `rsi.example.com` 加一条策略：只放行你自己的邮箱（邮箱验证码登录，免费版够用）。

## 注意

- **opentdx 出站**：若服务器在海外，通达信行情/akshare 可能受限或慢。
- **数据持久化**：`data/` 缓存随项目目录走，已是相对路径，无需额外处理。
- **opentdx 版本**：服务器从 PyPI 取最新版；若需锁定版本，在 `pyproject.toml` 的
  `dependencies` 里写成 `opentdx==x.y.z` 即可。
